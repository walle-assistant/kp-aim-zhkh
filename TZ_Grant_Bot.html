⚙️ ТЕХНИЧЕСКОЕ ЗАДАНИЕ

AI-Ассистент для автоматизации отчетности по грантам

1. ОПИСАНИЕ ПРОБЛЕМЫ

Получатели грантов (например, Фонда президентских грантов) сталкиваются со сложностью:

  • Отслеживание контрольных точек по этапам проекта
  • Сбор и систематизация материалов (фотографии, документы, ссылки)
  • Заполнение аналитической отчетности
  • Интеграция финансовых данных

Процесс отчетности — ручной, требует работы с множеством документов и папок.

2. РЕШЕНИЕ

Интерактивный AI-ассистент (чат-бот), который:

  • Помогает заявителям реализовать грант
  • Автоматизирует сбор и систематизацию материалов
  • Заполняет контрольные точки
  • Генерирует аналитические отчеты
  • Подключается к облачному хранилищу заявителя

3. ОСНОВНЫЕ ФУНКЦИИ

3.1 Управление грантом

  • Загрузка договора Фонда президентских грантов
  • Парсинг контрольных точек и сроков из договора
  • Напоминания о сроках и требуемых действиях

3.2 Сбор материалов

  • Прием файлов (фото, документы, ссылки) в чат
  • Автоматическая загрузка в облачное хранилище заявителя
  • Категоризация материалов по мероприятиям

3.3 Систематизация данных

  • Автоматическое создание структуры папок по мероприятиям
  • Сортировка материалов по типам (фото, документы, ссылки)
  • Сохранение в форматах: PDF, Google Docs, Markdown

3.4 Аналитическая отчетность

  • Автоматическое формирование отчета из собранных материалов
  • Заполнение контрольных точек
  • Экспорт отчета в требуемом формате

3.5 Финансовая интеграция (опционально)

  • Подключение API банка (Сбер, Тинькофф и т.д.)
  • Автоматический подтяг транзакций
  • Синхронизация с бюджетом гранта

4. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

4.1 Архитектура

  • Backend: Python/Node.js (обработка документов, бизнес-логика)
  • Chat Interface: Telegram / WhatsApp / Custom Web
  • Storage: Google Drive API / Yandex.Disk API
  • Processing: GPT-4 / Claude для парсинга и анализа

4.2 Интеграции

  • Google Drive API: создание папок, загрузка файлов, создание Google Docs
  • Yandex.Disk API: аналогично для Яндекс.Диска
  • Bank API (опционально): Сбер, Тинькофф для финансовых данных
  • OAuth: аутентификация пользователей

4.3 Обработка документов

  • OCR для сканов договоров
  • Структурированное извлечение данных (контрольные точки, сроки, требования)
  • Валидация данных из договора

4.4 Безопасность

  • Шифрование чувствительных данных
  • Ограничение доступа к грантовой информации
  • GDPR-compliant обработка персональных данных

5. ПОЛЬЗОВАТЕЛЬСКИЙ ПОТОК

Этап 1: Подготовка

  1. Заявитель регистрируется в сервисе
  2. Подключает Google Drive / Yandex.Disk
  3. Загружает договор с Фондом в чат

Этап 2: Парсинг гранта

  1. Бот парсит договор
  2. Извлекает контрольные точки, мероприятия, сроки
  3. Создает структуру папок в облачном хранилище

Этап 3: Реализация проекта

  1. Заявитель скидывает фотографии, документы, ссылки в чат
  2. Бот автоматически загружает в соответствующую папку
  3. Категоризирует материалы по мероприятиям

Этап 4: Отчетность

  1. По сроку бот напоминает о необходимости отчета
  2. Предполагает заполнить контрольные точки
  3. Генерирует черновик аналитического отчета
  4. Заявитель редактирует и утверждает

Этап 5: Экспорт

  1. Финальный отчет в формате PDF/Word/Google Docs
  2. Загрузка в Фонд (если поддерживается)

Технические

❓ Q3: Какие форматы договоров поддерживать? (PDF, Word, сканы)

Определить приоритеты OCR

❓ Q4: Как обеспечить аутентификацию в Google Drive / Yandex.Disk?

OAuth 2.0, но требуется тестирование refresh tokens

❓ Q5: Как бот определит, к какому мероприятию относится загруженный материал?

  • Вариант 1: Пользователь указывает вручную
  • Вариант 2: Бот анализирует по контексту и метаданным
  • Вариант 3: Гибридный подход (предложить, пользователь подтвердит)
